# AGENTS.md

## 这是什么

绿颊锥尾鹦鹉知识库的 Logseq graph，Markdown 格式。不是代码仓库。

## 全局约束

- **单一品种**：全库只围绕绿颊锥尾鹦鹉，不得出现其他鸟种属性或内容
- **文件格式**：Markdown，标题即页面名称（不含路径层级），`/` 在文件名中转为 `___`（见 `logseq/config.edn` `:file/name-format`）
- **属性驱动**：用 block 属性标记类型，不在标题中写层级前缀。`property-pages/enabled?` 为 `true`
- **属性排除列表**：`logseq/config.edn` 的 `:property-pages/excludelist` 已排除的非页面属性不得在内容中使用：`:duration` `:author` `:created` `:类型` `:推荐程度` `:主要营养` `:存在形式` `:化学性质` `:来源` `:毒性`
- **块缩进模式**：页面属性用 `key:: value` 置于文件顶部（不可加 `-` / `*` 前缀，不可用 `---` 分隔），正文中的 `##` 标题以 `- ##` 形式作为父块，标题内所有内容缩进 2 空格成为其子块
- **别名属性**：页面有别名（中/英文等）时，必须使用 `alias::` 属性置于文件头部，多个别名用英文逗号 `, ` 分隔，不加 `[[]]` 双链。正文中不再出现 `- 别名: xxx` 行
- **图标属性**：每个页面应使用 `icon::` 属性设置 emoji 图标，置于属性区。图标选取原则：饮食按食材本体选择（🥕🍎🥦等），疾病按症状/严重度选择（🤢☠️⚠️等），营养成分按富含食物选择（🍊等）
- **及时补充关联页面**：创建或编辑页面时，正文中使用的 `[[]]` 双链引用的关联概念（营养成分、疾病、身体构造、生理周期等），必须同步按对应 Schema 创建或补充内容，确保知识库双向链接完整

## 页面格式示例（完整）

```
type:: [[饮食]]
alias:: Apple
icon:: 🍎
安全等级:: [[安全]]
食物类型:: [[水果]]
年龄阶段:: [[全年龄段]]

- ## 基本信息
  - 常见品种: 红富士、嘎啦、金冠
  - 可食用部位: 果肉、果皮（充分清洗后）
- ## 营养成分
  - 突出营养: [[维生素C]]、[[膳食纤维]]
  - 含水量高（约 85%），适合夏季补水
- ## 喂食指南
  - 频率: 每日可少量给予
  - 份量: 每次 1/8~1/6 个苹果
  - 处理: 去核去籽
- ## 注意事项
  - 绝对去籽，种子含氰苷 → [[氰化物中毒]]
- ## 关联疾病
  - 过量: [[腹泻]]
```

**格式要点：**
- 属性区（文件头 3-6 行）紧贴顶部，无缩进、无前缀符号、无 `---` 分隔
- 每个 `## 标题` 前面加 `- ` 使其成为块，标题文字不含括号前缀
- 标题下的所有内容行统一缩进 2 空格（一级子块）
- 进一步嵌套在正文行下时再加 2 空格（共 4 空格，二级子块）
- 连续内容层级之间不留空行
- 段落与段落标题之间也不留空行

**完整示例（含多级嵌套）：**

```
type:: [[饮食]]
安全等级:: [[安全]]
食物类型:: [[加工食品]]
年龄阶段:: [[全年龄段]]

- ## 基本信息
  - 定义: Chop 是多种新鲜蔬果切碎混合分装冷冻的混合日粮
  - 一次批量制作，分装冷冻保存，解冻即喂
- ## 营养成分
  - Chop 的营养取决于所选配方，理想配比为:
    - 深色叶菜（30-40%）: [[羽衣甘蓝]]、[[菠菜]]
    - 其它蔬菜（30-40%）: [[胡萝卜]]、[[西兰花]]、[[南瓜]]、[[辣椒]]
    - 谷物/豆类（10-15%）: [[藜麦]]、[[绿豆芽]]
    - 水果（5-10%）: [[苹果]]、[[蓝莓]]
```

## 六种页面类型与属性 Schema

每条页面必须设置 `type::` 属性，值为以下六种双链之一，其它属性按对应 Schema 必填或可选。

### `type:: [[饮食]]` — 可喂食的食物条目

```

- ## 营养成分
- ## 喂食指南
- ## 注意事项
- ## 关联疾病
```

## 交叉链接规则

- 在正文中使用 `[[双链]]` 引用其他页面，不重复声明已有属性的内容
- 关联方向要点：
  - [[饮食]] → 链接到 [[营养成分]]、[[疾病]]（过量/缺乏导致）
  - [[营养成分]] → 链接到 [[饮食]]（食物来源）、[[身体构造]]（功能靶器官）、[[疾病]]（缺乏/过量）
  - [[疾病]] → 链接到 [[饮食]]、[[营养成分]]、[[身体构造]]、[[行为]]（症状表现）
  - [[行为]] → 链接到 [[疾病]]（医学病因）、[[生理周期]]（周期影响）
  - [[身体构造]] → 链接到 [[疾病]]、[[生理周期]]、[[营养成分]]
  - [[生理周期]] → 链接到 [[营养成分]]、[[行为]]、[[饮食]]、[[疾病]]

## 查询模式

用 `{{query}}` 按属性聚合页面，示例：

```
{{query (property type [[疾病]]) (property 严重程度 [[急症]]) }}
{{query (property type [[饮食]]) (property 安全等级 [[有毒]]) }}
{{query (property type [[饮食]]) [[换羽期]] }}
{{query (property type [[生理周期]]) }}
```

## 页面创建 Checklist

新建页面时确认：
1. [ ] `type::` 属性已设置且值为六种双链之一
2. [ ] 该 type 的 Schema 属性已全部填写；如有中/英文别名，使用 `alias:: Alias1, Alias2` 置于头部（不加 `[[]]`）
3. [ ] 属性值都使用了 `[[]]` 双链（除数值型如 `传染性:: 否`、`英文::`、`持续时间::`）
4. [ ] 正文至少建立了 2 个以上跨 type 链接
5. [ ] 未使用 `:property-pages/excludelist` 中的属性名
6. [ ] `##` 标题均以 `- ##` 块形式出现，内容统一缩进 2 空格
7. [ ] 属性区与 `- ##` 块之间无空行，块与块之间无空行
